# Plan: Modal Design System + Skill Expansion

## Context

The NPS module has 5 distinct modals across 4 files. The base Dialog/AlertDialog components already have good default styling (`bg-surface border-border/50 rounded-2xl shadow-2xl`), but the internal elements (labels, inputs, info cards, section headers) are inconsistent between files and don't follow the design system. The skill (`design-frontend`) explicitly says "Manter todos os modais inalterados" (rule #6), which needs to be replaced with a proper modal refactoring guide.

### Design system reference patterns (from other modules):

- **DialogContent**: `bg-card border-border` (standard override)
- **Info cards inside modals**: `bg-surface border border-border rounded-xl` (from mapas module)
- **Labels**: `text-xs text-text-muted font-medium`
- **Inputs**: `h-11 rounded-xl border border-border bg-input-bg px-4 text-sm text-text-main focus:ring-2 focus:ring-accent/40`
- **Section headers**: `text-sm font-semibold text-accent`
- **AlertDialog cancel**: `border-border text-text-main rounded-xl`
- **AlertDialog action (destructive)**: `bg-destructive hover:bg-destructive/90 text-white border-0 rounded-xl`
