# Plano: Módulo Rotas de Visitas

## Visão Geral

Módulo completo de planejamento e execução de rotas de visitas para consultores. O consultor planeja rotas diárias/semanais/mensais a partir de sua base de clientes, executa as rotas com rastreamento de geolocalização, e registra visitas com formulário pós-visita (reutilizando o CRM).

## 1. Banco de Dados (Migration SQL)

### Tabelas a criar:

#### `rotas_config` — Configuração do módulo por empresa
```sql
CREATE TABLE rotas_config (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  empresa_id UUID NOT NULL REFERENCES empresas(id) ON DELETE CASCADE UNIQUE,
  valor_km_reembolso DECIMAL(10,2) NOT NULL DEFAULT 0.00,
  raio_permitido_metros INT NOT NULL DEFAULT 300,
  created_at TIMESTAMPTZ NOT NULL DEFAULT now(),
  updated_at TIMESTAMPTZ NOT NULL DEFAULT now()
);
```

#### `rotas_clientes_base` — Base de clientes para rotas (populada via CSV ou importação)
```sql
CREATE TABLE rotas_clientes_base (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  empresa_id UUID NOT NULL REFERENCES empresas(id) ON DELETE CASCADE,
  usuario_id UUID NOT NULL REFERENCES auth.users(id) ON DELETE CASCADE,
  nome TEXT NOT NULL,
  telefone TEXT,
  cidade TEXT,
  estado TEXT,
  bairro TEXT,
  rua TEXT,
  numero TEXT,
  cep TEXT,
  endereco_completo TEXT,
  latitude DECIMAL(10,7),
  longitude DECIMAL(10,7),
  ticket_medio DECIMAL(10,2),
  categoria TEXT,
  ultima_visita DATE,
  fonte TEXT NOT NULL DEFAULT 'csv' CHECK (fonte IN ('csv', 'cadastros', 'crm')),
  fonte_id UUID,
  ativo BOOLEAN NOT NULL DEFAULT true,
  created_at TIMESTAMPTZ NOT NULL DEFAULT now(),
  updated_at TIMESTAMPTZ NOT NULL DEFAULT now()
);
```

#### `rotas` — Planejamento de rotas
```sql
CREATE TABLE rotas (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  empresa_id UUID NOT NULL REFERENCES empresas(id) ON DELETE CASCADE,
  usuario_id UUID NOT NULL REFERENCES auth.users(id) ON DELETE CASCADE,
  titulo TEXT NOT NULL,
  data_rota DATE NOT NULL,
  tipo TEXT NOT NULL CHECK (tipo IN ('diaria', 'semanal', 'mensal')),
  status TEXT NOT NULL DEFAULT 'planejada' CHECK (status IN ('planejada', 'em_execucao', 'realizada', 'nao_realizada', 'cancelada')),
  data_inicio TIMESTAMPTZ,
  data_fim TIMESTAMPTZ,
  local_inicio JSONB,
  local_fim JSONB,
  total_visitas INT DEFAULT 0,
  total_km DECIMAL(10,2) DEFAULT 0,
  total_tempo_trajeto_min INT DEFAULT 0,
  valor_reembolso DECIMAL(10,2) DEFAULT 0,
  created_at TIMESTAMPTZ NOT NULL DEFAULT now(),
  updated_at TIMESTAMPTZ NOT NULL DEFAULT now()
);
```

#### `rotas_clientes` — Clientes na rota (com ordem e status)
```sql
CREATE TABLE rotas_clientes (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  empresa_id UUID NOT NULL REFERENCES empresas(id) ON DELETE CASCADE,
  rota_id UUID NOT NULL REFERENCES rotas(id) ON DELETE CASCADE,
  cliente_base_id UUID NOT NULL REFERENCES rotas_clientes_base(id) ON DELETE CASCADE,
  ordem INT NOT NULL DEFAULT 0,
  status TEXT NOT NULL DEFAULT 'pendente' CHECK (status IN ('pendente', 'em_trajeto', 'em_visita', 'visitado', 'nao_visitado')),
  created_at TIMESTAMPTZ NOT NULL DEFAULT now(),
  updated_at TIMESTAMPTZ NOT NULL DEFAULT now(),
  UNIQUE(rota_id, cliente_base_id)
);
```

#### `rotas_trajetos` — Registro de cada trajeto (ponto A → ponto B)
```sql
CREATE TABLE rotas_trajetos (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  empresa_id UUID NOT NULL REFERENCES empresas(id) ON DELETE CASCADE,
  rota_id UUID NOT NULL REFERENCES rotas(id) ON DELETE CASCADE,
  rota_cliente_id UUID NOT NULL REFERENCES rotas_clientes(id) ON DELETE CASCADE,
  origem_lat DECIMAL(10,7),
  origem_lng DECIMAL(10,7),
  destino_lat DECIMAL(10,7),
  destino_lng DECIMAL(10,7),
  distancia_km DECIMAL(10,2),
  duracao_minutos INT,
  valor_reembolso DECIMAL(10,2),
  data_inicio TIMESTAMPTZ,
  data_fim TIMESTAMPTZ,
  created_at TIMESTAMPTZ NOT NULL DEFAULT now()
);
```

#### `rotas_visitas` — Registro de cada visita
```sql
CREATE TABLE rotas_visitas (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  empresa_id UUID NOT NULL REFERENCES empresas(id) ON DELETE CASCADE,
  rota_id UUID NOT NULL REFERENCES rotas(id) ON DELETE CASCADE,
  rota_cliente_id UUID NOT NULL REFERENCES rotas_clientes(id) ON DELETE CASCADE,
  cliente_base_id UUID NOT NULL REFERENCES rotas_clientes_base(id) ON DELETE CASCADE,
  consultor_id UUID NOT NULL REFERENCES auth.users(id),
  data_inicio TIMESTAMPTZ NOT NULL,
  data_fim TIMESTAMPTZ,
  duracao_minutos INT,
  local_inicio JSONB,
  local_fim JSONB,
  dentro_raio BOOLEAN DEFAULT true,
  formulario JSONB NOT NULL DEFAULT '{}',
  created_at TIMESTAMPTZ NOT NULL DEFAULT now()
);
```

### Índices e RLS:
- Índices em: `rotas(usuario_id, data_rota)`, `rotas_clientes(rota_id)`, `rotas_trajetos(rota_id)`, `rotas_visitas(rota_id)`
- RLS: policies de SELECT/INSERT/UPDATE/DELETE para `authenticated` (padrão do projeto)
- Triggers `updated_at` nas tabelas `rotas_config`, `rotas_clientes_base`, `rotas`, `rotas_clientes`

  opcoes TEXT[] DEFAULT '{}',
  obrigatorio BOOLEAN NOT NULL DEFAULT true,
  ordem INT NOT NULL DEFAULT 0,

## 5. Fluxo de Execução

### Planejamento:
1. Consultor acessa `/rotas`
2. Clica "Nova Rota" → Modal com opções:
   - **Manual**: Selecionar clientes um a um
   - **Por Filtros**: Cidade, ticket médio, última visita, categoria
   - **Automática**: Sistema seleciona baseado nos filtros + prioridade
3. Define: título, data, tipo (diária/semanal/mensal)
4. Salva → Rota criada com status `planejada`

### Execução:
1. Consultor clica "Iniciar Rota" → Grava data/hora/localização
2. Lista de clientes aparece na ordem definida
3. Para cada cliente:
   - **Iniciar Trajeto**: Abre Google Maps/Waze com best route
   - **Iniciar Visita**: Só se estiver a ≤300m do destino (verifica geolocalização)
   - Grava distância e duração do trajeto (Google Distance Matrix ou cálculo Haversine)
   - Calcula reembolso = distância_km × valor_km_reembolso
   - **Finalizar Visita**: Só se estiver a ≤300m + iniciou visita
   - Timer contabiliza duração da visita
   - Modal com formulário pós-visita (obrigatório)
   - Item marcado como "visitado"
4. Quando todos visitados → Botão "Finalizar Rota"
5. Grava data/hora/localização fim + totais

### Validações:
- Iniciar visita: só se geolocalização ≤300m do destino
- Finalizar visita: só se visitou E está ≤300m
- Visitas não finalizadas após data da rota → status `nao_visitado`
- Rotas não executadas → status `nao_realizada`

### Formulário Pós-Visita (Dinâmico):
- Administrador configura as perguntas do formulário
- Tipos de pergunta: texto curto, texto longo, data, múltipla escolha, seleção (select), rádio
- Schema armazenado em `rotas_form_perguntas` (similar ao `form_schema` existente)
- Respostas salvas como JSON no campo `rotas_visitas.formulario`
- Renderização dinâmica do formulário baseado no schema

## 6. Integrações

### Google Maps / Waze (Deep Links):
```typescript
// Abrir Google Maps
`https://www.google.com/maps/dir/?api=1&origin=${lat},${lng}&destination=${destLat},${destLng}`

// Abrir Waze
`https://www.waze.com/ul?ll=${destLat},${destLng}&navigate=yes`
```

### Geolocalização (Browser API):
```typescript
navigator.geolocation.getCurrentPosition(success, error, {
  enableHighAccuracy: true,
  timeout: 10000,
  maximumAge: 0
});
```

### Cálculo de Distância:
- **Opção 1 (Recomendada)**: Google Maps Distance Matrix API — precisa de API key
- **Opção 2 (Fallback)**: Fórmula de Haversine — sem custo, menos preciso
- Implementar as duas opções, com fallback automático
