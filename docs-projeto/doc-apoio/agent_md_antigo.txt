## 🛡️ POLÍTICA DE SOBREVIVÊNCIA DE COTA (OBRIGATÓRIO)
Este repositório opera sob restrição severa de tokens. O descumprimento destas regras resulta em suspensão da cota de 5h.

### 1. Orquestração e Agregação (MCP & Skills)
- **NÃO liste diretórios recursivamente**: Use a ferramenta MCP de busca semântica ou `grep` específico.
- **Skill-First**: Antes de qualquer tarefa complexa, verifique se há uma skill em `.agents/skills/`. Se existir, carregue APENAS o `SKILL.md`.
- **MCP Aggregation**: Para dados do banco, use funções de agregação do servidor MCP Supabase. Proibido ler tabelas inteiras para "entender o esquema".

### 2. Gerenciamento de Contexto Ativo (Atendimento ao Cache)
- **NÃO ignore o Prompt Caching**: Mantenha as instruções de sistema estáveis. Não peça mudanças de "personalidade" no meio da sessão.
- **Lazy Loading**: O agente deve ler arquivos apenas quando o plano de ação (Step 4) for aprovado. 
- **Context Clearing**: Ao finalizar uma etapa (ex: Etapa 6 concluída), o agente deve sugerir ao usuário o comando `/clear` para resetar a janela de tokens.

### 3. Técnicas de Prompting "Lean" (Baixo Consumo)
- **Estilo Caveman**: Respostas ultra-curtas. Ex: "Código atualizado. Teste passando." em vez de explicações pedagógicas.
- **ReWOO (Reasoning Without Observation)**: Crie o plano completo de passos *antes* de usar qualquer ferramenta. 
- **Edição Cirúrgica**: Proibido `rewrite_file`. Use apenas ferramentas de substituição de linhas específicas para evitar reenviar o arquivo todo no output.

### 4. Alternativas de Infraestrutura (Burlar Cota)
- **Roteamento Headroom**: Priorize `Gemini-1.5-Flash` para 90% das tarefas.
- **Fallback Externo**: Se o erro "Quota Exceeded" ocorrer, solicite ao usuário a ativação da `External API Key` (Google AI Studio) configurada no ambiente.
- **Offline First**: Use ferramentas de análise estática locais (via MCP) antes de enviar código para revisão na nuvem.
