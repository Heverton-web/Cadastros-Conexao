Arquitetura & Estrutura

Organização do projeto, fluxo de navegação, registro de módulos e estrutura de diretórios.

Frameworks e Bibliotecas

O ERP Conexão adota uma arquitetura moderna com TanStack Start (React Router) para roteamento SSR, Vite como build tool, Supabase como backend (PostgreSQL + Auth + Storage) e Tailwind CSS v4 para estilização. O gerenciamento de estado é feito com Zustand, e o sistema de ícones utiliza Lucide React.

Sistema de Registro (Registry)

O coração da arquitetura modular é o sistema de Registry em src/registry/. Ele permite que cada módulo se auto-registre com suas rotas, permissões, itens de navegação e ações, de forma desacoplada.

// Exemplo de registro de módulo registerModule({ key: "cadastros-conexao", nome: "Cadastros", descricao: "Gestão de cadastros de profissionais", icon: Users, routes: ["/clientes", "/consultor", "/relatorios", ...], ambientes: ["cadastro", "consultor"], permissions: ["ver_todos_cadastros", "aprovar_cadastro", ...] });

Fluxo de Navegação

  1. Login — Rota / → autenticação via Supabase Auth
  2. AuthGuard — Em _auth.tsx, verifica sessão, carrega perfil, permissões e módulos ativos da empresa
  3. AppLayout — Renderiza Header + NavSidebar + Outlet + BottomNav
  4. Seleção de Módulo — NavSidebar filtra itens por módulo selecionado e permissões do usuário
  5. Renderização — <Outlet /> exibe o conteúdo da rota ativa

Rotas Públicas vs Autenticadas

O sistema possui 6 rotas públicas (fora do layout autenticado) e mais de 45 rotas protegidas (dentro de _auth).

Rotas Públicas: / (login), /pre-cadastro/$token, /nps/survey, /linktree/$id, /hub/cliente/dashboard/$empresaId, /crm/aceitar-convite/$token

Server Functions e Middleware

Funções server-side (ex: src/lib/visitas.functions.ts) são protegidas pelo middleware requireSupabaseAuth e utilizam esquemas Zod para validação de entrada. O tratamento de erros SSR é customizado em src/start.ts e src/server.ts.

Ambientes

Cada módulo pode ser habilitado para ambientes específicos:

  • cadastro — Módulos operacionais de cadastro
  • consultor — Módulos voltados ao consultor de campo
  • tecnologia — Módulos de administração técnica
  • suporte — Módulos de suporte ao cliente
  • ambos — Módulos transversais
