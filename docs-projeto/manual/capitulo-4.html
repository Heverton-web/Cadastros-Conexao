Permissionamento Granular

86+ permissões distribuídas em 8 módulos, controle de acesso por grupo, ambiente e Super Admin.

Visão Geral do Sistema

O ERP Conexão possui um sistema de permissões granular e modular. Cada ação ou página do sistema pode ser controlada individualmente. As permissões são armazenadas como um flat Record<string, boolean> e são carregadas no AuthProvider durante o login.

Usuários com is_super_admin = true recebem automaticamente TODAS as permissões do sistema, sem necessidade de configuração individual.

Estrutura de Dados

Tipo: Permissoes = Record<string, boolean> • ModuloAcesso: { acessar: boolean; paginas: string[]; acoes: string[] } • ModulosAcesso: Record<string, ModuloAcesso>

Como as Permissões são Aplicadas

1. Registro Centralizado

Cada módulo registra suas permissões no registry através da função registerPermission(), informando key, label, descrição e grupo.

2. AuthProvider

No login, o AuthProvider carrega as permissões do usuário e as armazena como um objeto plano. O Super Admin ganha acesso total automaticamente:

Super Admin: Se profile.is_super_admin === true, todas as permissões são atribuídas como true.

3. Navegação Filtrada

O hook useNavItems() em src/components/layout/useNavItems.ts filtra os itens de navegação com base nas permissões do usuário. Cada NavItemRegistration possui uma função permissionCheck que determina se o item deve aparecer.

4. Controle em Páginas

Individualmente, cada página pode verificar permissões específicas para liberar ou ocultar funcionalidades, botões e seções.

Grupos de Permissões (Cadastros - Exemplo)

  • Escopo de Dados — ver_todos_cadastros
  • Visualização — ver_relatorios, visualizar_documento
  • Aprovação de Cadastro — aprovar_cadastro, reprovar_cadastro, solicitar_correcao_cadastro
  • Aprovação de Documentos — aprovar_documento, reprovar_documento, solicitar_correcao_documento
  • Aprovação de Campos — aprovar_campo, reprovar_campo, solicitar_correcao_campo
  • Credenciais — gerenciar_credenciais, gerenciar_credenciais_admin
  • Administração — excluir_cadastro, gerenciar_config, gerar_links

Ambientes vs Permissões

Além das permissões individuais, cada usuário possui um ambiente que define seu contexto de atuação. Um usuário do ambiente consultor pode ter permissões diferentes de um usuário do ambiente cadastro, mesmo dentro da mesma empresa. Os ambientes são:

  • cadastro — Acesso operacional aos cadastros
  • consultor — Acesso às ferramentas de campo (CRM, visitas)
  • tecnologia — Acesso técnico e de configuração
  • suporte — Acesso limitado para suporte
  • ambos — Acesso a múltiplos contextos
