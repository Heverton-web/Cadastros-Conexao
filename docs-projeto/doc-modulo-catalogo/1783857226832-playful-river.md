# Plano: Rota de Design do Catálogo (`/catalogo/admin/design`)

## Resumo

Criar uma rota exclusiva onde Admin e Super Admin personalizam toda a aparência da loja catálogo por empresa. Super Admin vê seletor de empresas. Os dados são salvos em nova tabela `catalogo_design_config` (JSONB flexível). O `StoreLayout` carrega a config do banco e aplica CSS variables dinamicamente, sobrescrevendo os defaults do `theme.css`.

**Preview em tempo real**: Mockup de celular (padrão NPS/Pré-cadastro) com miniatura da loja, aplicando CSS vars instantaneamente via inline styles.

## Etapa 1: Migration SQL (Supabase)

Criar tabela `catalogo_design_config`:

```sql
CREATE TABLE catalogo_design_config (
  empresa_id UUID PRIMARY KEY REFERENCES empresas(id) ON DELETE CASCADE,
  config JSONB NOT NULL DEFAULT '{}',
  updated_at TIMESTAMPTZ DEFAULT now()
);

-- RLS: empresa vê apenas sua own config
ALTER TABLE catalogo_design_config ENABLE ROW LEVEL SECURITY;

CREATE POLICY "empresa_select_own" ON catalogo_design_config
  FOR SELECT USING (empresa_id = (
    SELECT empresa_id FROM profiles WHERE id = auth.uid()
  ));

CREATE POLICY "empresa_upsert_own" ON catalogo_design_config
  FOR ALL USING (empresa_id = (
    SELECT empresa_id FROM profiles WHERE id = auth.uid()
  ));

-- Super admin pode tudo
CREATE POLICY "super_admin_all" ON catalogo_design_config
  FOR ALL USING (
    EXISTS (SELECT 1 FROM profiles WHERE id = auth.uid() AND is_super_admin = true)
  );
```

### Lógica de Acesso

```tsx
function AdminDesignPage() {

  const isSuperAdmin = profile?.is_super_admin === true

  const [config, setConfig] = useState(DEFAULT_CATALOGO_CONFIG)
  const [saving, setSaving] = useState(false)

  // Super Admin: carrega lista de empresas
  useEffect(() => {

  useEffect(() => {

  }, [selectedEmpresaId])

  // Salvar

    await saveCatalogoDesign(selectedEmpresaId, config)
    toast.success("Design salvo!")
  }

  return (
    <AdminLayout>

        {isSuperAdmin && <EmpresaSelector empresas={empresas} value={selectedEmpresaId} onChange={setSelectedEmpresaId} />}

      {/* Grid 2 colunas: edição + preview */}

        </div>
      </div>
    </AdminLayout>
  )

```

### Como Funciona o Preview em Tempo Real

1. **State compartilhado**: `config` (CatalogoDesignConfig) é state do componente pai `AdminDesignPage`
2. **Cada aba de edição** recebe `config` e `setConfig` — ao alterar qualquer campo, o state atualiza
3. **`PhoneMockup` → `StoreMiniatura`** recebe o `config` como prop e aplica via `style={cssVars}` — atualização instantânea
4. **Sem re-render pesado**: o preview é leve (~80 linhas de JSX), atualização occurs via React state
5. **Celular sticky**: no desktop, o preview fica fixo à direita enquanto o admin edita à esquerda

Adicionar `useEffect` que:

```tsx
useEffect(() => {
  async function loadDesign() {
    if (!empresaId) return
    const config = await getCatalogoDesign(empresaId)
    const merged = mergeWithDefaults(config)

    const el = document.querySelector('.catalogo-theme')
    if (!el) return

    const cssMap: Record<string, string> = {

      '--color-accent-fg': merged.colors.accentFg,
      '--color-bg': merged.colors.bg,

      '--color-surface-hover': merged.colors.surfaceHover,

      '--color-text-main': merged.colors.textMain,
      '--color-text-muted': merged.colors.textMuted,
      '--color-border-subtle': merged.colors.borderSubtle,
      '--color-input-bg': merged.colors.inputBg,
      '--color-input-border': merged.colors.inputBorder,
      '--color-success': merged.colors.success,
      '--color-error': merged.colors.error,
    }
    for (const [k, v] of Object.entries(cssMap)) {
      (el as HTMLElement).style.setProperty(k, v)
    }

    // Efeitos: blobs
    if (merged.effects.enableBlobs) { ... }

    // Logos
    if (merged.images.logoUrl) { setLogoUrl(merged.images.logoUrl) }

    setDesignConfig(merged)  // expor para filhos via context ou state
  }
  loadDesign()
}, [empresaId])
```

**Importante:** O `theme.css` continua como fallback — quando não há config salva no DB, os defaults do CSS são usados. A config do DB só sobrescreve quando existe.
